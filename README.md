# 🔋 ChargeRL: 사용자 패턴 기반 배터리 수명 보호 강화학습 시뮬레이션

본 프로젝트는 스마트기기 사용자의 불규칙한 충전 패턴을 학습하여, **배터리 노화(Aging)를 최소화**하고 **사용자 편의성(완충 상태 확보)**을 동시에 달성하기 위한 AI 에이전트 개발 프로젝트입니다.

## 1. 연구 배경 및 목적
대부분의 사용자는 취침 시 충전기를 연결하고 아침에 분리합니다. 이 과정에서 배터리가 100%에 도달한 후에도 수 시간 동안 고전압 상태로 방치되는 **'과충전 및 고전압 유지'** 문제가 발생하여 배터리 수명이 단축됩니다.

- **문제점:** 기존의 즉시 충전 방식(**Greedy Policy**)은 완충 후 기기 해제 시점까지 배터리를 고전압 상태로 방치하여 화학적 노화를 가속함.
- **해결책:** 강화학습(Q-Learning)을 통해 사용자의 기기 해제 시점을 예측하고, 그 직전에 충전을 완료하는 **'최적 대기 시점'**을 도출함.

## 2. 핵심 기술적 특징

### 2.1 UX 중심의 보상 설계 (UX-Aware Reward Shaping)
단순한 수치 최적화를 넘어, 사용자가 기기를 들었을 때 배터리가 부족해 겪는 불편함을 방지하는 데 초점을 맞췄습니다.
- **완충 성공 보상:** 기기 해제 시점(Unplug Time)에 SoC(충전 상태)가 **99.5% 이상**일 경우 높은 보상을 부여합니다.
- **방치 시간 감점 (Aging Penalty):** 100% 도달 후 해제 시점까지 흐른 시간(Idle Time)만큼 감점을 부여하여, 에이전트가 충전 완료 시점을 최대한 해제 시점에 맞추도록 유도합니다.
- **물리적 한계 고려:** 초기 배터리 잔량이나 충전 시간이 절대적으로 부족한 상황에서는 감점을 최소화하여 학습의 왜곡을 방지했습니다.

### 2.2 현실적인 시뮬레이션 환경 (Realistic Environment)
- **State Definition:** 요일(Day), 현재 시간(Hour), 현재 잔량(SoC), 사용자 활동 패턴(Pattern).
- **Dynamic Pattern:** 평일과 주말의 기기 해제 시점이 다른 현실적인 패턴을 반영하여 에이전트가 상황별로 유연하게 대처하도록 구현했습니다.

### 2.3 학습 최적화 및 안정화
- **State Discretization:** 연속적인 시간과 SoC 데이터를 구간화(Quantization)하여 Q-Table의 수렴 속도를 높였습니다.
- **Adaptive Epsilon & Alpha Decay:** 학습이 진행됨에 따라 탐험(Exploration)과 학습률(Learning Rate)을 점진적으로 낮추어, 수렴 이후의 그래프 진동을 억제하고 안정적인 정책을 확보했습니다.

## 3. 시뮬레이션 결과 (Performance Analysis)

본 시뮬레이션은 Q-Learning 에이전트와 기존 정책들(즉시 충전, 고정 대기 등)을 5,000 에피소드 동안 비교 실험하였습니다.

| 비교 항목 | Greedy Policy (즉시 충전) | Fixed Policy (2시간 대기) | Q-Learning (Proposed AI) |
| :--- | :--- | :--- | :--- |
| **배터리 보호** | **최악 (방치 시간 매우 김)** | 보통 (일괄적 대기) | **최상 (해제 직전 완충)** |
| **사용자 만족도** | 높음 (항상 완충) | 불안정 (충전 부족 발생) | **매우 높음 (완충 유지)** |
| **패턴 대응** | 불가 (단순 반복) | 불가 (고정 방식) | **가능 (요일/패턴별 적응)** |

> **결과 요약:**
> Greedy 방식은 배터리를 장시간 100% 상태로 방치하여 노화를 유발하는 반면, 
> **Q-Learning 에이전트는 사용자의 기상 시간을 학습하여 충전 시점을 뒤로 미룸으로써 배터리 건강 상태(SoH)를 획기적으로 보호합니다.**
